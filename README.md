## סקירה כללית
MedTracker היא אפליקציית מובייל-first (עברית, RTL) לניהול הטיפול התרופתי של אדם מבוגר.
בן המשפחה המטפל רואה במבט אחד מה נלקח היום, מה הבא בתור, מתי המלאי עומד להיגמר —
וכן מדדים, יומן תסמינים, מידע חירום (ICE) ועוזר AI, במקום אחד.

## הבעיה
בני משפחה מנהלים תרופות של הורה מבוגר בראש, בפתקים או בוואטסאפ: לא ברור אם התרופה
נלקחה, מתי נגמר המרשם, ומה לומר לרופא. התוצאה — נטילות שנשכחות וחוסר שקט. MedTracker
הופך את זה לתמונה אחת ברורה ונגישה.

## קהל יעד
בן/בת משפחה מטפל/ת (המשתמש העיקרי), מטפל/ת מקצועי/ת, והמבוגר עצמו (מצב נגישות:
טקסט גדול, ניגודיות, יעדי מגע גדולים).

## מתחרים ובידול
| פתרון קיים | החיסרון | הבידול שלנו |
|---|---|---|
| פתקים / זיכרון | נשכח, אין מעקב | מעקב נטילות מתועד + תזכורות |
| וואטסאפ משפחתי | אין מבנה/סטטוס | מצב נטילה לכל תרופה + עמידות ורצף |
| אקסל / יומן ידני | מסורבל, לא נגיש | UI נגיש RTL, מבט אחד |
| אפליקציות גלובליות | אנגלית, לא RTL | עברית מלאה, ICE עם 101, מותאם ישראל |

**בידול ליבה:** עברית/RTL + נגישות לקשישים + שכבת AI (מידע ואינטראקציות) + מידע חירום —
בחבילה אחת ממוקדת.

## הזרימה המרכזית
הרשמה/התחברות → הוספת תרופה (שם, כמות, מינון, שעות) → סימון "לקחתי" → מעקב עמידות,
מלאי והיסטוריה → (אופציונלי) שאלת עוזר ה-AI.

## משתמש דמו
מצב הדגמה: אפשר להירשם עם כל אימייל (סיסמה 6+ תווים). בכניסה ראשונה נוצרים מטופל
ברירת-מחדל ו-3 תרופות דמו אוטומטית, כך שהזרימה ניתנת לבדיקה מיד.

## ארכיטקטורה
- **Frontend:** React + Vite + TailwindCSS, RTL. פרוס ב-Vercel.
- **Backend:** Supabase — PostgreSQL (מודל מנורמל), Auth (אימייל + Google), RLS,
  ו-Edge Function שמסתירה את מפתח ה-AI.
- **AI:** Anthropic Claude דרך ה-Edge Function בלבד.

### מודל הנתונים
ERD: [`ERD.md`](./ERD.md) · DDL מלא עם RLS: [`schema.sql`](./schema.sql).

### שירותים חיצוניים ואינטגרציות
| שירות | סוג | למה משמש |
|---|---|---|
| Supabase Auth | אוטנטיקציה | הרשמה/התחברות (אימייל + Google OAuth) |
| Supabase PostgreSQL | בסיס נתונים | מטופלים, תרופות, נטילות, מדדים, יומן, חירום |
| Supabase RLS | אבטחת גישה | מטופל נחשף רק למטפל המורשה |
| Supabase Edge Function | לוגיקת שרת | קריאה מאובטחת ל-Claude (הסתרת מפתח) |
| Anthropic Claude API | AI | מידע על תרופות, אינטראקציות, תובנה יומית |
| Vercel | דיפלוי | אירוח ה-Frontend + CI מ-GitHub |

## הרצה מקומית
```bash
npm install
cp .env.example .env.local      # מלא VITE_SUPABASE_URL ו-VITE_SUPABASE_ANON_KEY
npm run dev
```

## הגדרת Supabase (פעם אחת)
1. צור פרויקט ב-supabase.com → SQL Editor → הרץ את `schema.sql`.
2. Authentication → Providers → הפעל Email (ו-Google ל-OAuth).
3. Project Settings → API → העתק URL ו-anon key אל `.env.local`.
4. (ל-AI) `supabase functions deploy claude-proxy` ו-`supabase secrets set ANTHROPIC_API_KEY=...`
   (הקוד ב-`supabase/functions/claude-proxy/index.ts`).

## פריסה ב-Vercel
push ל-GitHub → Vercel → New Project (Vite) → הגדר `VITE_SUPABASE_URL` ו-`VITE_SUPABASE_ANON_KEY` → Deploy.
ב-Supabase → Auth → URL Configuration הוסף את דומיין ה-Vercel.
